# IP 定址（IP Addressing）

## 什麼是 IP 位址？

每一台連上網路的設備（電腦、手機、平板）都會被分配一個 IP 位址。
IP 從你的 ISP（Internet Service Provider，網路服務提供商）取得。

> **筆記重點**
> 任何一台裝置——電腦、手機或平板——都有能力扮演伺服器的角色。只要連上網路，作業系統就會自動為它掛上一個 IP 位址，而這個位址是由你的 ISP（網路服務提供商）所配發的。

## 網路型態

| 類型 | 全名 | 範圍 | 用途 |
|------|------|------|------|
| **LAN** | Local Area Network | 辦公室、家庭 | 連接同一空間的設備、共用印表機 |
| **MAN** | Metropolitan Area Network | 城市、學區 | 大學、醫院之間的連線 |
| **WAN** | Wide Area Network | 跨國、全球 | 透過電話線、海底光纜連接世界 |

## IPv4 vs IPv6

### IPv4

- 使用 32-bit，格式：`192.168.1.1`（四組十進位數字）
- 總共約 **43 億**（$2^{32}$）個位址
- 問題：地址已耗盡，全球 80 億人口 + IoT 設備遠超此數量

### IPv6

- 使用 128-bit，格式：`2001:0db8:85a3::1`（八組十六進位）
- 總共 **340 澗**（$2^{128}$）個位址，趨近無限
- 額外優勢：內建 IPsec 加密、自動設定（SLAAC）、更適合行動裝置

## 公有 IP vs 私有 IP

| 類型 | 範圍 | 說明 |
|------|------|------|
| **私有 IP** | `10.x.x.x` / `172.16.x.x` / `192.168.x.x` | 家庭或公司內部使用 |
| **公有 IP** | 其他 | 全球唯一，ISP 分配 |

**NAT（Network Address Translation）**：路由器將多台私有 IP 設備的請求，
轉換為同一個公有 IP 對外發出。就像大樓的「分機」對應到「總機電話」。

---

## 靜態 IP vs 動態 IP（DHCP）

**DHCP（Dynamic Host Configuration Protocol）** 自動分配 IP，採「租約」機制：

1. **Discover**：設備大喊「誰可以給我 IP？」
2. **Offer**：DHCP 伺服器（通常是路由器）提供一個 IP
3. **Request**：設備確認要這個 IP
4. **Acknowledge**：伺服器確認，租約通常 24 小時

**靜態 IP**：適用於伺服器、印表機等需要固定地址的設備。

---

## MAC 位址 vs IP 位址

| | MAC 位址（L2） | IP 位址（L3） |
|-|--------------|--------------|
| **比喻** | 身分證字號（永久不變） | 住家地址（隨網路改變） |
| **範圍** | 同一區域網路內 | 跨網路遠距傳輸 |
| **格式** | `00:0c:29:4f:aa:bb` | `192.168.1.1` |

---

## 子網路遮罩（Subnet Mask）

`255.255.255.0` 告訴電腦：IP 前三組相同的設備是「鄰居」，可以直接溝通；
不在範圍內的封包則送往**預設閘道（Default Gateway）**，也就是路由器。

## 前端工程師視角

- `localhost` 和 `127.0.0.1` 都指向自己的電腦（Loopback 位址）
- 看到 `169.254.x.x` 代表 DHCP 失敗，無法上網（APIPA）
- 公司內部 API（`192.168.x.x`）回家切換 4G 後就連不到，是因為不在同一子網路
