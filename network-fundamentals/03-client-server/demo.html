Client-Server 架構 Demo

Client-Server 基本架構

💻
Client（瀏覽器）
發出 HTTP 請求
→ GET /index.html →
← 200 OK HTML ←
🖥
Server（伺服器）
監聽 Port 80/443

DNS 解析流程動畫

輸入網址後，瀏覽器在開始下載 HTML 前，必須先完成 DNS 解析

Port Number（連接埠）

Port協定用途說明
80HTTP一般網頁沒有加密，瀏覽器預設連此埠
443HTTPS安全網頁SSL/TLS 加密連線
22SSH遠端伺服器命令列連線到伺服器
3306MySQL資料庫後端程式連接資料庫
3000 / 8080Dev Server前端開發npm run dev 通常在這裡

預設資源：index.html

以下三種 URL 最終都回傳同樣的 index.html

https://example.com
https://example.com/
https://example.com/index.html
伺服器的「預設文件」設定 → 沒有路徑時自動找 index.html 回傳。

Chrome DevTools Timing 對照 OSI

DevTools 顯示對應 OSI 層代表意義優化方向
DNS Lookup L7 應用層（DNS） 網域→IP 的解析時間 用 dns-prefetch、改換更快的 DNS
Initial Connection L4 傳輸層（TCP） TCP 三向握手時間 用 CDN 讓伺服器離使用者更近
SSL L6 展示層（TLS） 加密握手時間 HTTP/3 的 0-RTT 可消除這段
TTFB L3-L7（端對端） 等待伺服器第一個位元組 後端 API 速度 / 資料庫優化
Content Download L1-L4（傳輸） 下載檔案本體 壓縮圖片、Minify、Gzip
📌 開啟 Chrome DevTools → Network → 點選任一請求 → Timing 分頁，就可以看到這張圖的實際數字。
