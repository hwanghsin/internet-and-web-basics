TCP vs UDP 對比

📦 TCP
傳輸控制協定
  • 保證送到（重傳機制）
  • 保證順序
  • 需要握手（建立連線）
  • 流量控制
HTTP, HTTPS, Email, FTP
🚀 UDP
使用者資料包協定
  • 發了就不管（不重傳）
  • 不保證順序
  • 無連線（快速）
  • 低延遲
DNS 查詢, 影片串流, 遊戲, HTTP/3
比喻：TCP = 掛號信（有確認收到）；UDP = 平信（發出去就算了）
HTTP/3 用 UDP 但在上面自己實作了可靠性（QUIC 協定）

HTTP 版本演進

版本傳輸層多路復用握手時間換網路前端影響
HTTP/1.1 TCP ❌ 需排隊 多次往返 斷線重連 需要 CSS Sprite、合併檔案
HTTP/2 TCP ✅ 同一連線 普通 斷線重連 多個小請求不再是問題
HTTP/3 UDP (QUIC) ✅ 獨立流 0-RTT（瞬間） ✅ 無縫切換 Wi-Fi↔4G 不斷線

HTTP/1.1 vs HTTP/2 連線模型

HTTP/1.1（隊頭阻塞）
同域 6 條 TCP 連線，每條排隊
HTTP/2（多路復用）
一條 TCP 連線，並行傳輸

WebSocket：從 HTTP 升級到全雙工
