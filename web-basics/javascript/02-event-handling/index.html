事件傳播：冒泡（Bubble）vs 捕獲（Capture）

addEventListener 的第三個參數（布林值）控制傳播方向：
false（預設）= 冒泡（由內向外） |  true = 捕獲（由外向內）

點擊按鈕觀察傳播順序：
BODY
OUTER DIV
INNER DIV
（點選上方按鈕設定模式，再點 BUTTON）
// 冒泡（預設）：button → inner → outer → body
el.addEventListener('click', handler, false);

// 捕獲：body → outer → inner → button
el.addEventListener('click', handler, true);

// 傳播順序公式：
// 1.body(capture) → 2.outer(capture) → 3.inner(capture)
// → 4.button(capture) → 5.button(bubble)
// → 6.inner(bubble) → 7.outer(bubble) → 8.body(bubble)

Event Delegation（事件委派）

不在每個子元素上綁定事件，改在父元素監聽，透過 event.target 判斷實際點擊目標。
優點：動態新增的元素自動有事件；節省記憶體。

  • 項目 1 [刪除]
  • 項目 2 [編輯]
  • 項目 3 [查看] 
// ✅ 事件委派：只在 ul 上綁定一次
ul.addEventListener('click', (e) => {
  const li = e.target.closest('li'); // 找到最近的 li 祖先
  if (!li) return;
  const action = li.dataset.action;
  const id     = li.dataset.id;
  console.log(`${action} item ${id}`);
});

// ❌ 避免這樣：每個 li 各綁一次（浪費記憶體，動態新增的 li 沒有事件）

自訂事件（CustomEvent）

// 建立並觸發自訂事件
const event = new CustomEvent('user:login', {
  detail: { user: 'Alice', role: 'admin' },  // 攜帶的資料
  bubbles: true,     // 是否冒泡
  cancelable: true,  // 是否可 preventDefault
});
document.dispatchEvent(event);

// 監聽自訂事件
document.addEventListener('user:login', (e) => {
  console.log('使用者登入：', e.detail.user);
});
