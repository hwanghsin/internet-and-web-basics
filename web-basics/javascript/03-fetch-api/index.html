🌐 網路概念橋接
這個頁面是 web-basics（JS 實作）和 network-fundamentals（網路概念）的交會點。
每次你呼叫 fetch('https://api.example.com/data')，背後走完了整個 OSI 七層：
L7 DNS → 把 api.example.com 解析成 IP（參考：03-client-server/）
L4 TCP → SYN / SYN-ACK / ACK 三向握手（參考：05-tcp-ip-model/）
L6 TLS → HTTPS 加密協商（參考：04-osi-model/）
L7 HTTP → Method + Path + Headers + Body（參考：02-request-response/）
L7 HTTP → Status Code + Response Body 回傳

fetch() 背後的網路流程

GET / POST 實際請求（使用 httpbin.org）

GET 請求（取得資料）
↑ 這個 GET 請求會經過 DNS 查詢 httpbin.org → TCP 連線 → HTTP/1.1 或 HTTP/2
POST 請求（送出資料）
↑ POST body 以 JSON 格式傳送（Content-Type: application/json），對應 L6 展示層的序列化
→ Request（發出）

Promise 鏈 vs async/await 寫法比較

// ❶ 原始 Promise 鏈（.then() 串接）
fetch('https://api.example.com/users')
  .then(response => {
    if (!response.ok) throw new Error(`HTTP ${response.status}`);
    return response.json();        // 解析 JSON（L6 展示層：反序列化）
  })
  .then(data => {
    console.log('使用者數量：', data.length);
  })
  .catch(err => {
    console.error('請求失敗：', err.message);
  })
  .finally(() => {
    console.log('請求完成（無論成功失敗）');
  });


// ❷ async/await（更易讀，等價於上面）
async function loadUsers() {
  try {
    const response = await fetch('https://api.example.com/users');

    // response.ok 為 false 代表 4xx / 5xx（fetch 不會 throw）
    if (!response.ok) throw new Error(`HTTP ${response.status}`);

    const data = await response.json();
    console.log('使用者數量：', data.length);
  } catch (err) {
    console.error('請求失敗：', err.message);
  } finally {
    console.log('請求完成');
  }
}
⚠️ fetch() 只有在網路完全斷線（DNS / TCP 失敗）時才會 reject。
4xx / 5xx 的 HTTP 錯誤不會 reject，需要自己檢查 response.ok 或 response.status。

Headers 分頁
